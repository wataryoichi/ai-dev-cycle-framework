# Feature Ideas

## Near-term
- [ ] Codex stdout pipe: `codex ... | devcycle turbo --title "..."`
- [ ] `turbo --watch` — re-run on file changes
- [ ] `rollback --dry-run` — preview before reverting
- [ ] `stable` tag — mark known-good versions
- [ ] Auto CHANGELOG.md generation from turbo history
- [ ] Dual JSON files alongside Markdown (request.json, review.json, etc.)

## Medium-term
- [ ] Claude Code hook: auto-trigger review after implementation
- [ ] `devcycle init` — bootstrap new project with config
- [ ] Dashboard / TUI for cycle status
- [ ] PyPI publish
- [ ] `devcycle diff --from v1 --to v2`

## Longer-term
- [ ] Full Claude↔Codex loop without human intervention
- [ ] Web UI
- [ ] Team features: shared index, review assignment
- [ ] Metrics: cycle time, findings trend, rollback frequency
